# 5. Экспериментальные исследования

## 5.1 Цели и план экспериментов

Цель экспериментального этапа — количественно оценить точность и устойчивость разработанной модели локализации дефектов, а также сравнить ее с базовыми подходами.

Основные задачи:
1. Проверка базовой точности на тестовой выборке.
2. Оценка устойчивости к шумам.
3. Оценка чувствительности к объему обучающих данных.
4. Сравнение с классическими/базовыми ML-методами.

## 5.2 Организация экспериментальных данных

### 5.2.1 Состав выборок

Используются:
- тренировочная выборка для настройки параметров модели;
- валидационная для выбора гиперпараметров;
- независимая тестовая для финальной оценки.

Разделение выполняется по сценариям эксперимента, чтобы исключить повтор идентичных режимов в разных подвыборках.

### 5.2.2 Сценарии повреждений

Рассматриваются сценарии:
- дефект вблизи узла;
- дефект в пролете элемента;
- различная глубина/интенсивность повреждения;
- разные уровни внешнего шума.

## 5.3 Результаты базового эксперимента

По итогам базового теста анализируются:
- средняя евклидова ошибка локализации;
- MAE по каждой координате;
- распределение ошибок по зонам.

Типичные наблюдения:
- наибольшая точность достигается при достаточной плотности датчиков;
- ошибки возрастают в зонах со слабой наблюдаемостью;
- комбинированные признаки (время + спектр + метаданные) дают лучший результат, чем изолированные группы признаков.

Результаты представляются в виде:
- сводной таблицы метрик;
- boxplot-диаграмм по сценариям;
- карты ошибок по конструктивной схеме.

## 5.4 Устойчивость к шумам и неопределенностям

Проверяется влияние добавленного гауссова шума, пропусков каналов и малых смещений частоты дискретизации. Для каждого уровня помех оценивается деградация метрик:

$$
\Delta_{noise}=\frac{\mathrm{MAE}_{noise}-\mathrm{MAE}_{base}}{\mathrm{MAE}_{base}}\cdot 100\%.
$$

Вывод по устойчивости формируется по двум критериям:
- скорость ухудшения точности;
- наличие катастрофических выбросов ошибки.

## 5.5 Чувствительность к объему данных

Проводится обучение на подвыборках разного объема (например, 30%, 50%, 70%, 100% от полного train-набора) с последующей оценкой на фиксированном тесте.

Ожидаемая зависимость:
- при росте объема данных ошибка уменьшается;
- после определенного порога наблюдается эффект насыщения.

Результат позволяет определить минимально достаточный объем данных для промышленной эксплуатации.

## 5.6 Сравнение с альтернативными методами

Для сопоставления рассматриваются:
- базовый MLP без свёрточного блока;
- классические регрессионные методы (например, Random Forest/Gradient Boosting по агрегированным признакам);
- инженерно-эвристическая локализация по модальным индикаторам.

Сравнение проводится по:
- точности;
- устойчивости;
- времени расчета;
- интерпретируемости для эксперта.

## 5.7 Анализ ошибок модели

Типичные источники ошибок:
- близко расположенные дефекты с похожими сигнатурами;
- недостаточная информативность датчиков для отдельных зон;
- выход параметров режима за рамки обучающего диапазона;
- недостаточная представленность редких сценариев в train-наборе.

Меры снижения ошибок:
- расширение датасета сложными сценариями;
- усиление регуляризации;
- введение ансамблирования;
- калибровка доверия к прогнозам.

## 5.8 Практическая интерпретация результатов

Даже при сопоставимых средних метриках ключевое значение имеет доля крупных промахов локализации. Для эксплуатационного применения рекомендуется совместная интерпретация:
- координатного прогноза;
- доверительного показателя;
- инженерных ограничений (запрет физически невозможных позиций дефекта).

## 5.9 Выводы по главе

Экспериментальные исследования подтверждают работоспособность предложенного подхода и его преимущество над базовыми альтернативами по совокупности критериев. Модель демонстрирует практическую пригодность при условии корректной подготовки данных и регулярного обновления на новых сценариях эксплуатации.

## 5.10 Дополнительная проработка: протокол воспроизводимого эксперимента

Для каждой серии экспериментов фиксируются:
- версия датасета;
- версия модели и конфигурации;
- random seed;
- вычислительная среда;
- набор метрик и критерии остановки.

Протокол серии:
1. Подготовка подвыборок.
2. Обучение модели.
3. Оценка на test.
4. Анализ устойчивости.
5. Формирование отчета.

## 5.11 Дополнительная проработка: статистическая проверка различий

При сравнении методов рекомендуется не ограничиваться средними значениями метрик, а применять статистические критерии значимости.

Для парных сравнений по сериям запусков можно использовать:
- t-тест для связанных выборок (при близкой нормальности);
- критерий Уилкоксона (непараметрический вариант).

Интерпретация:
- нулевая гипотеза: отличий между методами нет;
- при $p < 0.05$ различие считается статистически значимым.

## 5.12 Дополнительная проработка: анализ надежности в инженерных терминах

Для практической эксплуатации вводятся показатели:
- доля попаданий в допустимую зону погрешности;
- доля критических промахов;
- среднее время обнаружения и локализации дефекта.

Пусть допустимая ошибка равна $\Delta_{adm}$. Тогда:

$$
P_{adm}=\frac{1}{n}\sum_{i=1}^{n} I(\Delta_i \le \Delta_{adm}),
$$

где $I(\cdot)$ — индикаторная функция.

Высокое значение $P_{adm}$ непосредственно связано с пригодностью модели для эксплуатационных решений.

## 5.13 Дополнительная проработка: интерпретация отрицательных результатов

Если в отдельных сериях наблюдается деградация, это не является «неудачей» исследования, а указывает на границы применимости модели. Необходимо:
- зафиксировать условия деградации;
- проверить состав признаков;
- оценить достаточность обучающей выборки;
- предложить корректирующие мероприятия.

Такой подход повышает научную добросовестность и практическую ценность итоговой работы.

## 5.23 Дополнительная проработка: экспериментальная оценка модальных методов

Для повышения интерпретируемости результатов нейросетевой локализации в экспериментальный контур включена проверка трех модальных подходов:
- анализ форм колебаний;
- индекс повреждений;
- частотный анализ собственных форм колебаний.

### 5.23.1 Анализ форм колебаний в эксперименте

На каждом тестовом сценарии вычисляется MAC между эталонными и текущими модальными формами:

$$
MAC_i=\frac{|(\phi_i^0)^T\phi_i^d|^2}{\left((\phi_i^0)^T\phi_i^0\right)\left((\phi_i^d)^T\phi_i^d\right)}.
$$

Критерий интерпретации:
- $MAC_i \to 1$ - состояние близко к эталонному;
- устойчивое снижение $MAC_i$ по нескольким модам - признак структурного изменения.

### 5.23.2 Индекс повреждений в эксперименте

Для узлов/зон конструкции рассчитывается индекс повреждений:

$$
DI_j=\sum_{i=1}^{m}\frac{\left|\kappa_{i,j}^{d}-\kappa_{i,j}^{0}\right|}{\max\limits_{r}\left|\kappa_{i,r}^{0}\right|+\epsilon}.
$$

Экспериментально анализируются:
1. Корреляция максимума $DI_j$ с истинной зоной дефекта.
2. Устойчивость ранжирования зон по $DI_j$ при шумовых возмущениях.
3. Согласованность карты $DI_j$ с прогнозом ML-модели.

### 5.23.3 Частотный анализ собственных форм в эксперименте

По каждой моде определяется относительный частотный сдвиг:

$$
\delta f_i=\frac{f_i^{0}-f_i^{d}}{f_i^{0}}.
$$

Интегральный показатель:

$$
FI=\sum_{i=1}^{m} w_i\,\delta f_i,\quad \sum_{i=1}^{m}w_i=1.
$$

В серии экспериментов оценивается чувствительность $\delta f_i$ и $FI$ к глубине повреждения и уровню шума, а также их вклад в итоговую ошибку локализации.

1. Конструкция фиксируется в граничных условиях консольного типа.
2. На выбранной позиции задается дефектный сценарий (масса/положение груза).
3. Выполняется отклонение стержня на малую величину и свободный отпуск.
4. Регистрируется сигнал затухающих колебаний.
5. Отдельно записывается шумовой прогон без возмущения для оценки базового фона.

Для повышения устойчивости модели используются повторные прогоны по каждой позиции и последующее усреднение/фильтрация признаков.

### 5.24.3 Практические наблюдения при сборе данных

- увеличение массы имитатора дефекта повышает различимость сценариев в сигнале;
- для малых масс различия между соседними позициями могут быть статистически незначимыми;
- неинформативные каналы (с низкой межклассовой вариативностью) целесообразно исключать из обучающего набора;
- шумовые записи улучшают устойчивость модели к помехам при эксплуатации.

### 5.24.4 Влияние на качество локализации

Включение стендовых серий с шумовыми прогонами и вариацией массы позволяет:
- снизить долю ложных срабатываний;
- повысить стабильность модальных индикаторов ($MAC$, $DI$, $FI$);
- улучшить обобщающую способность модели на новых сценариях.

Данный протокол рекомендуется использовать как базовый регламент пополнения датасета при дальнейшем развитии системы.

Ты работаешь с backend приложения для расчёта стержневой конструкции и предсказания дефектов.

Ниже текущая логика системы:

1. FEM-расчёт

- `apps/backend/app/services/fem.py`: собирается глобальная матрица жёсткости стержней, решается система `K*u = F`.
- На выходе считаются:
  - перемещения узлов;
  - модуль вектора смещения узла;
  - напряжения в стержнях через осевую деформацию и `E`.

2. API

- `apps/backend/app/api/routes.py`: все запросы нормализуются в `quasi-static` режим.
- Используются шаги нагрузки `0.25 / 0.5 / 0.75 / 1.0`.
- FEM запускается на каждом шаге.

3. Дефекты

- `apps/backend/app/services/defects.py`: дефекты не пересчитывают механику заново.
- Они усиливают базовые напряжения через коэффициенты концентрации:
  - трещина,
  - коррозия,
  - уменьшение сечения и т.д.

- Формула:
  `sigma_new = sigma_base * Π(Kt)`

4. AI-модель

- `apps/backend/app/services/nn.py`
- Сейчас модель предсказывает:
  - `risk_by_rod` для каждого стержня в диапазоне `0..1`;
  - топ-3 самых рискованных стержня;
  - вероятную позицию дефекта вдоль стержня через heatmap-сегменты.

- Признаки сейчас:
  - длина / стройность стержня;
  - площадь сечения;
  - относительная нагрузка;
  - prior по дефектам;
  - учёт уже заведённых дефектов.

- Если есть обученные коэффициенты — использовать линейную модель + sigmoid.
- Если нет — использовать эвристику.
- Также возвращается качество (`precision / recall / f1`) относительно текущих отмеченных дефектов.

Нужно изменить логику расчётов и предсказаний.

Новая постановка:

- Возле каждого узла стоит датчик.
- Каждый датчик измеряет смещение узла по двум координатам:
  - `dx` — по оси X,
  - `dy` — по оси Y.

- Эти измерения должны стать основной входной информацией для модели предсказания дефектов.
- Модель должна использовать не только расчётные значения FEM, но и реальные/наблюдаемые данные с датчиков.
- Идея: по картине смещений в узлах (`dx`, `dy`) определять вероятность дефекта, наиболее подозрительные стержни и примерную зону дефекта.

Что нужно реализовать:

1. Изменить модель данных / входные данные API

- Добавить возможность передавать показания датчиков по узлам.
- Формат должен быть примерно таким:

  ```json
  {
    "node_sensors": {
      "node_id_1": {"dx": ..., "dy": ...},
      "node_id_2": {"dx": ..., "dy": ...}
    }
  }
  ```

- Если датчик для узла отсутствует, нужно корректно обрабатывать пропуск:
  - либо брать FEM-значение как fallback,
  - либо помечать sensor availability отдельным признаком.

2. Изменить FEM / постобработку

- Сохранить текущий FEM-расчёт.
- Помимо модуля смещения узла, явно возвращать компоненты смещения:
  - `ux`
  - `uy`

- Для каждого узла в результирующих структурах должны быть доступны:
  - расчётный `ux`, `uy`;
  - модуль смещения;
  - при наличии — измеренный `dx`, `dy`;
  - residual / ошибка между расчётным и измеренным:
    - `rx = dx - ux`
    - `ry = dy - uy`
    - `r_norm = sqrt(rx^2 + ry^2)`

3. Изменить логику AI-модели

- Перестроить признаки так, чтобы модель опиралась на датчики возле узлов.
- Для каждого стержня формировать признаки на основе двух его узлов:
  - `dx_i`, `dy_i`, `dx_j`, `dy_j`
  - `ux_i`, `uy_i`, `ux_j`, `uy_j`
  - residuals на концах стержня
  - разность смещений между концами
  - относительное удлинение / деформация, оценённая через смещения узлов
  - длина стержня, площадь, стройность, относительная нагрузка
  - наличие ранее заведённых дефектов

- Добавить идею аномальности:
  - чем сильнее наблюдаемые смещения расходятся с FEM-ожиданием, тем выше вероятность дефекта.

- Если есть обученные коэффициенты:
  - обновить feature vector;
  - использовать линейную модель + sigmoid.

- Если коэффициентов нет:
  - сделать новую эвристику, завязанную в первую очередь на:
    - величину residual;
    - локальный скачок смещений между соседними узлами;
    - аномально высокую деформацию стержня;
    - наличие уже известных дефектов.

4. Изменить предсказания

  - `top_3_rods`
  - probable defect position / heatmap

- Позицию дефекта вдоль стержня оценивать так:
  - если аномалия сильнее выражена со стороны одного узла — смещать вероятность ближе к нему;
  - если аномалия примерно симметрична — дефект ближе к центру;
  - вернуть это в виде heatmap по сегментам вдоль стержня.
